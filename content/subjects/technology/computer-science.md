# Computer Science Learning Path

Computer Science is the study of computation, algorithms, and information. This guide provides a structured path from complete beginner to job-ready software engineer.

## üéØ Learning Objectives

By following this path, you will:
- Master at least one programming language (Python or JavaScript recommended)
- Understand fundamental data structures and algorithms
- Build complete applications
- Prepare for technical interviews
- Contribute to open-source projects

## üß† Phase 2: Data Structures & Algorithms (Months 3-4)

### Goals
- Understand Big O notation
- Master arrays, linked lists, stacks, queues
- Learn trees, graphs, hash tables
- Implement sorting and searching algorithms

### Resources

**Courses:**
- [Data Structures (UC San Diego)](https://www.coursera.org/learn/data-structures) on Coursera
- [Algorithmic Toolbox](https://www.coursera.org/learn/algorithmic-toolbox) on Coursera
- [CS61B](https://inst.eecs.berkeley.edu/~cs61b/) - UC Berkeley Data Structures

**Books:**
- "Grokking Algorithms" by Aditya Bhargava
- "Introduction to Algorithms" (CLRS) - The bible of algorithms
- "Cracking the Coding Interview" by Gayle Laakmann McDowell

**Visualization:**
- [VisuAlgo](https://visualgo.net/) - Algorithm visualizations
- [Algorithm Visualizer](https://algorithm-visualizer.org/)

### Practice Platforms
- [LeetCode](https://leetcode.com/) - 2000+ interview problems
- [HackerRank](https://www.hackerrank.com/) - Structured practice
- [AlgoExpert](https://www.algoexpert.io/) - Curated problems with video explanations

### Key Algorithms to Master
1. **Sorting:** Quick sort, Merge sort, Heap sort
2. **Searching:** Binary search, BFS, DFS
3. **Graph:** Dijkstra's, Bellman-Ford, A*
4. **Dynamic Programming:** Fibonacci, Knapsack, LCS

## ‚ùì FAQ

**Q: Do I need a CS degree?**
A: No, but you need to demonstrate skills. A portfolio of projects is often more valuable.

**Q: Which language should I start with?**
A: Python (easiest) or JavaScript (most versatile). Both have massive communities.

**Q: How long until I get a job?**
A: 6-12 months of dedicated study for most people.

**Q: Is it too late to start?**
A: Absolutely not. People start at 30, 40, 50+ and succeed.

## üåü Final Advice

1. **Consistency beats intensity** - Code every day, even if just 30 minutes
2. **Build projects** - Don't just watch tutorials
3. **Read code** - Study open source projects
4. **Teach others** - Explain concepts to solidify understanding
5. **Stay curious** - Technology changes; continuous learning is key

**Remember:** Every expert was once a beginner. Start today.
